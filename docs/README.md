# Methodology

The sanitized, reusable version of the method behind the starters. These docs are the *why* and *when*; the starter repos are the *how*.

*Population in progress — the documents below are being adapted from MAS's internal methodology into general, client-neutral form.*

- **`adoption-journey.md`** — how a nonprofit moves from "we should do something with AI" to owning and evolving its own assistant. Four stages: public workshop → client workshop → build → sustain.
- **`productionalization-model.md`** — taking an assistant from prototype to a self-sufficient, client-owned production system, including the non-technical-operator handoff loop (runbook → coding agent → PR → preview → merge).
- **`platform-and-subscription-guide.md`** — choosing an office suite, an LLM, a CRM connection, and a coding agent, on a nonprofit budget. The decision tree the `np-ai-discovery` skill operationalizes.
