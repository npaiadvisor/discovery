# Contributing to the Nonprofit AI Commons

This is a public commons for nonprofit-AI practitioners. The goal is reciprocal: we publish production-grade patterns, and we invite peers to publish theirs alongside. Everyone's reach grows.

## Anyone can contribute

- **Open an issue** to propose a pattern, report a problem, or ask a question.
- **Open a pull request** to add or improve a pattern, connector, doc, or case study.
- No CLA. By submitting, you agree your contribution is offered under this repository's licenses (MIT for code, CC BY-SA 4.0 for docs — see `LICENSE` and `LICENSE-docs`).
- Contributions are attributed to their author and organization.

## Ground rules

1. **Pass the sanitization check.** Read [SANITIZATION.md](SANITIZATION.md). No client data, no credentials, no organization-internal schemas — ever, including in examples and fixtures.
2. **Be honest about cost and effort.** Case studies should state real cost/time/maintenance, not marketing numbers.
3. **Prefer mainstream, contractor-friendly tooling.** A nonprofit should never be locked to one consultant.
4. **Keep it self-sufficiency-first.** Patterns should leave the nonprofit more capable, not more dependent.

## Review

Initial maintainers (npaiadvisor + invited peers) review issues and PRs asynchronously. Decisions happen in the open on GitHub.
